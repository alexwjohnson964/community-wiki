# Code After Coffee Documentation

This wiki is built to document contribution guidelines and any tools/resources that would be usefull
for the community

## Sections

1. Git Commands: This section is dedicated to outlining useful git commands that will come up in the development process
    1. [remote](./git-commands/remote.md): Manage connections between local git and remote repositories (Github)
    2. [branch](./git-commands/branch.md): Manage branches in local git
    3. [fetch](./git-commands/fetch.md): Fetching changes from remote repositories
    4. [pull](./git-commands/pull.md): Pulling changes from remote repositories to local repositories
    5. [push](./git-commands/push.md): Pushing changes from local repositories to remote repositories
    6. [rebase](./git-commands/rebase.md): Managing commit history of local repositories
    7. [cherry-pick](./git-commands/cherry-pick.md): Adding specific pre-commited changes to local repositories

2. Git Commit Guidelines
    1. [feat](./git-commits/feat.md): Implemented a new feature
    2. [fix](./git-commits/fix.md): Fixed an existing feature
    3. [refactor](./git-commits/refactor.md): Refactoring existing code 
    4. [docs](./git-commits/docs.md): Documentation about existing code
    5. [chore](./git-commits/chore.md): Chores related to the management of the code
    6. [build](./git-commits/build.md): Changes to the build process of the code
    7. [ci](./git-commits/ci.md): Changes to client interaction of the code
    8. [style](./git-commits/style.md): Changes to the styling of code
    9. [test](./git-commits/test.md): Testing of exisiting code
    10. [Breaking Change (!)](./git-commits/breaking.md): If new changes breaks existing application infrastructure
